# 🌫️ VoidArchive v2.6 (Museum Preservation Engine)

A high-fidelity digital preservation system for the academic labour and legacy of **Sincere Bhattarai**. 

Developed with a focus on **Glossy Minimalism**, this archive serves as a view-only museum, ensuring institutional-grade preservation of PPTX, DOCX, and PDF documents through an automated archival pipeline.

---

## 🏛️ System Features (Museum Standard)

- **Glossy Minimalism Layout**: A refined, high-contrast studio aesthetic (VoidOS v2.6).
- **High-Fidelity Archival Engine**: Automated transformation of raw documents into sharp, optimized **WebP (120 DPI)** slide nodes.
- **Museum Discovery Panel**: A robust Search Protocol for querying archival nodes across all protocols.
- **Peer Review Integration**: An identity-verified rating system for archival feedback.
- **GitHub Preservation Pipeline**: Automated CI/CD processing of raw assets into servable archive bundles.

## 🛠️ Archival Protocol

1. **Upload**: Place raw files in `/assets/Raw`.
2. **Process**: Run the archival engine (`scripts/processor.py`).
3. **Artifacts**: Optimized WebP nodes are stored in `/assets/Processed`.
4. **Preservation**: Commit processed assets to GitHub for long-term accessibility.
